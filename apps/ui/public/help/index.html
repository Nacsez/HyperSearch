Overview

What HyperSearch Does

HyperSearch is an API-first local research service. It sends web queries to SearXNG, optionally fetches result pages, extracts text, and can ask a local model provider such as LM Studio to synthesize an answer with citations. The web UI is a first-party operator console on top of the same backend endpoints that headless clients use.

Search mode

Returns ranked search results quickly. This is the fastest path and should be your default when you want links, snippets, caching behavior, and basic summaries.

Research mode

Pulls top results, fetches page content, extracts readable text, and asks the active provider to synthesize an answer with citations and a debug trace.

Start Here

Quick Start

  • Open HyperSearch Desktop and select Start. The launcher waits until API, UI, SearXNG, Valkey, and Caddy are ready.
  • Select New Session after the stack reports ready.
  • Run a regular search first when you want links, snippets, and fetched source material.
  • Use Research Synthesis when you want HyperSearch to review collected sources and prepare a cited answer.
  • Use Save Session to write an XML session archive and place it in the Session Library.
  • Open Operations to review readiness, enable or disable LLM features, discover loaded models, and test the active provider.
Search-only use is valid:

HyperSearch can run without LM Studio or any other model provider. Search remains available, and research requests return a source review when model synthesis is off or unavailable.

Reference

Field Guide

Field Used By Meaning Operator Notes
Query Search, Research The search phrase sent to SearXNG. Use focused, factual queries first. Broader wording increases result variety but can increase noise.
Engines Search, Research Comma-separated engine list such as google,duckduckgo. Leave blank to use backend defaults. Restrict this when comparing engine behavior.
Categories Search, Research Comma-separated SearXNG categories such as general,news. Useful for isolating news-like results or narrowing broad searches.
Language Search, Research Language hint, for example en-US. Use when you need locale-sensitive ranking or local-language material.
Date Range Search, Research Freshness filter such as past day, week, month, or year when the selected search engine supports it. Useful for current events. Leave blank for general web discovery.
Results to Collect Search, Research Total number of search results HyperSearch should collect. The UI calculates backend page settings automatically. Increase this when you need deeper recall; larger values are slower and produce more local cache entries.
Research Sources Research How many collected search hits to fetch, summarize, and synthesize. Start around 3 to 5. Larger values are supported, but research can take substantially longer because sources are processed in batches.
Safe Search Search, Research SearXNG safety mode: off, moderate, or strict. Leave at moderate for most use.
Cache Policy Search, Research Controls cache usage for search, fetch, extract, and synthesis paths. use is normal. bypass is best for fresh verification. refresh forces recomputation and updates cache.
Dedupe Search Removes duplicate or near-duplicate results. Usually leave on unless you are analyzing engine overlap.
Fetch pages Search Fetches linked pages after search. Use this when snippets are not enough and you want body content attached to results.
Extract text Search Runs extraction on fetched pages. Best paired with Fetch pages. Useful when snippets are noisy or truncated.
Search summary Search Requests a compact search-level summary. Good for quick scans without invoking full research mode.
Provider Override Research Forces research synthesis through a specific configured provider. Leave blank to use the default provider. Useful for A/B comparison across providers later.
Timeout (ms) Search, Research Search collection budget and model-call guidance for long research requests. Use a larger value for heavy research. HyperSearch still uses provider-level timeouts so a local model can finish long synthesis work reliably.
Headless-only options:

The API also exposes options like streaming and include_debug_trace. These are useful when integrating HyperSearch into scripts, agents, or other services.

Workflow

Presets And Profiles

Presets store a reusable search and research request shape. Use them when you regularly repeat the same kind of search, source count, safety setting, provider choice, or cache policy.

  • Set the fields you want in the search form.
  • Click Save Preset.
  • Give the preset a descriptive name such as Recent news sweep or Wikipedia fact check.
  • Use the saved preset chips at the bottom of the search form to load it later.
  • After loading a preset, you can still change individual fields before running the next request.
  • Mark one preset as default when you want new sessions to open with that profile already selected.
Recommended naming pattern:

Use names that encode both intent and shape, for example News month fast, Deep research 5 pages, or English general bypass cache.

Export

Save Sessions

Save Session writes the current session as XML and adds it to the Session Library. The XML format is useful for durable personal archives, notebook ingestion, and downstream automation. Browser sessions download the XML through the browser. Embedded desktop sessions hand the XML to the desktop shell, which writes the file into the export folder configured in desktop settings.

File naming

Exported files use the first 48 characters of the active query, sanitized for Windows filenames, followed by a timestamp such as 20260426_153015.xml.

Included fields

Open Search Settings, then XML export contents, to choose whether the export contains metadata, the question, search results, fetched full text, summaries, research answer, research sources, activity log, and provider trace.

Privacy note:

XML exports can include scraped page text, local provider/model metadata, and detailed activity logs. Turn off fields you do not need before sharing an export outside your own machine.

Research Results

Reading Metrics

Search source counts

  • Search Sources Requested is the number of search results HyperSearch attempted to collect from SearXNG.
  • Search Sources Retrieved is the number of search results returned after paging, ranking, and deduplication.
  • A smaller retrieved value can happen when the search engine does not provide enough unique results for the request.

Research source counts

  • Research Sources Requested is the number of collected search hits HyperSearch attempted to fetch, extract, and review.
  • Research Sources Retrieved is the number of usable source documents included in the answer or source review.
  • Research Source Shortfall shows the gap when pages are blocked, empty, duplicated, or otherwise unavailable for extraction.
Citations and sources:

The numbered sources under Sources Used are the citation set for the research answer. HyperSearch uses that single source count in the metrics panel to avoid duplicate citation and document-count labels.

API

Headless Usage

Headless use goes through the same backend as the UI. Local requests on localhost do not need a token. LAN access must be enabled in the desktop launcher and paired clients should supply X-HyperSearch-Token.

PowerShell search example

.\scripts\Invoke-HyperSearch.cmd `
  -Action search `
  -Query "history of cincinnati ohio" `
  -Results 10 `
  -Summarize
{
  "query": "history of cincinnati ohio",
  "page": 1,
  "results_per_page": 5,
  "max_pages": 1,
  "safe_search": 1,
  "dedupe": true,
  "fetch_pages": false,
  "extract_text": false,
  "summarize": true,
  "streaming": false,
  "timeout_ms": 15000,
  "cache_policy": "use"
}

$body = @'
{
  "query": "history of cincinnati ohio",
  "page": 1,
  "results_per_page": 5,
  "max_pages": 1,
  "safe_search": 1,
  "dedupe": true,
  "fetch_pages": false,
  "extract_text": false,
  "summarize": true,
  "streaming": false,
  "timeout_ms": 15000,
  "cache_policy": "use"
}
'@

Invoke-RestMethod `
  -Uri http://127.0.0.1:8090/v1/search `
  -Method Post `
  -ContentType "application/json" `
  -Body $body

PowerShell research example

.\scripts\Invoke-HyperSearch.cmd `
  -Action research `
  -Query "origin of ronald mcdonald" `
  -Results 25 `
  -ResearchSources 5 `
  -TimeoutMs 60000

{
  "query": "origin of ronald mcdonald",
  "page": 1,
  "results_per_page": 10,
  "max_pages": 1,
  "top_n": 3,
  "timeout_ms": 60000,
  "cache_policy": "use",
  "provider": null,
  "streaming": false,
  "include_debug_trace": true
}
'@

Invoke-RestMethod `
  -Uri http://127.0.0.1:8090/v1/research `
  -Method Post `
  -ContentType "application/json" `
  -Body $body

Deploy

Installation And Deployment

Standard install

  1. Use the full installer media whenever you want the most predictable offline setup path.
  2. Run HyperSearch_1.0.0_x64-setup.exe from the media folder.
  3. If payload\images is present, setup loads Docker images locally with docker load.
  4. If no image archive is present, setup uses online mode and pulls prebuilt images.
  5. Review the setup result screen before closing the installer.
  6. Launch HyperSearch Desktop and press Start.
  7. Keep LAN mode disabled unless another local-network device needs access.

Developer startup

  1. Create a Python virtual environment and install the API package from apps/api.
  2. Install UI dependencies in apps/ui with npm install.
  3. Run the API with Uvicorn on 127.0.0.1:8000.
  4. Run the UI with npm run dev on 127.0.0.1:5173.
  5. For Compose release mode, run .\scripts\Deploy-HyperSearch.cmd -Action up.
  6. For local image builds, add -f docker-compose.yml -f docker-compose.dev.yml up -d --build.

Docker Desktop

  • Setup detects Docker Desktop and can guide installation when missing.
  • Startup requires a clean Docker engine version response.
  • The desktop launcher waits for API, UI/Caddy, SearXNG, Valkey, /v1/live, and search readiness before sessions open.
  • Online media requires registry/DNS/proxy access for image pulls.
  • Full media can start without registry access after images are loaded.

LM Studio and models

  • Search-only is supported. Research becomes source review when LLM features are off or unavailable.
  • Research synthesis requires a local OpenAI-compatible endpoint.
  • Docker-managed HyperSearch usually reaches LM Studio at http://host.docker.internal:1234.
  • Low-resource systems may be configured as search-only until a suitable model is selected.
  • Use Operations to toggle LLM features, discover loaded models, select a model, and test or verify it before saving the provider.
Network exposure:

HyperSearch is designed to bind to localhost by default. LAN mode is opt-in from the desktop launcher and paired browsers must send the generated pairing token. Do not expose HyperSearch to the public internet.

Diagnostics:

Installer logs, setup summaries, desktop logs, and full command logs are written under %LOCALAPPDATA%\HyperSearch\logs. Use desktop Settings > Export Diagnostics when you need a support bundle. Diagnostics redact token, key, password, auth, and credential values.

Support

Troubleshooting And Validation

When startup does not complete

  • Open the desktop launcher and select Refresh to rerun Docker and readiness checks.
  • Open Logs to review Docker Compose output from API, UI, Caddy, SearXNG, and Valkey.
  • Confirm Docker Desktop is running and that your Windows user can access the Docker engine.
  • Use Settings > Export Diagnostics if you need to share a redacted support bundle.

When research cannot synthesize

  • Open Operations and confirm LLM features are enabled if you want model synthesis.
  • Discover models from the provider endpoint, select the loaded model, then run Test or Verify.
  • If no model is available, continue in search-only mode. Research will return a source review instead of an error.
  • For large research runs, raise Timeout (ms) and expect HyperSearch to process sources in batches.
Useful validation loop:

Run a small search, save the session, run a small research request, increase the source counts, and then export diagnostics if anything fails. This confirms the stack, UI, source collection, model path, session saving, and logging path in one pass.

Most common operator loop

  1. Start the stack.
  2. Run search first.
  3. Inspect results.
  4. Run research when you need a cited answer or source review.
  5. Save the session when you want a durable archive.

Known Behaviors

Do not misread these as failures

  • Some pages on the web will block automated fetches with 403 responses.
  • Provider tests are usually slower than search because they hit the local model backend.
  • Research duration can increase sharply when source counts are high or pages are heavy.
  • Search-only mode is normal when no local model is configured.
